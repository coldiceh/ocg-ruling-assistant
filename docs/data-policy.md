# 数据与裁定策略

## 资料优先级

1. 官方数据库公开 Q&A、官方规则书、官方说明。
2. 事务局确认记录或赛事裁判确认记录。
3. 结构化镜像数据源，例如 YGOResources。
4. 玩家社区整理资料。
5. 模型推理或店内推定。

只有第 1、2 类，或能明确追溯到第 1 类的结构化镜像，才适合在界面中显示为“已确认”。第 4、5 类必须标记为“推定”或“待补出处”。

## 快照过期

`data/snapshot-meta.json` 中的 `generatedAt` 和 `freshnessDays` 控制页面置信度。

- 快照未生成：显示“种子资料”。
- 快照在有效期内：已确认条目可以显示较高置信度。
- 快照过期：即使命中已确认条目，也显示“需复核”。

游戏王 OCG 裁定可能变更，尤其是新卡发售、数据库 FAQ 更新、事务局口径调整后。过期快照不能作为唯一依据。

## 裁定变更处理

同步脚本会把来源 revision 或 changed manifest 写入 `snapshot-meta.json` 和 `sync-report.json`。如果发现 Q&A 内容变化，应：

1. 保留来源 ID、更新时间和旧结论摘要。
2. 把相关条目标记为需要复核。
3. 在答案中优先展示最新快照。
4. 对常见争议案例增加人工备注，说明“某日期前后处理不同”。

## 模型回答边界

接入大模型时，模型只能做三件事：

- 把自然语言问题拆成卡名、时点、连锁、状态。
- 检索资料并摘出相关依据。
- 根据依据生成解释。

模型不能在没有出处时把结论标记为确定裁定。
当前后端会把没有直接 Q&A/FAQ 支持的模型回答降级为“规则推理”或“不能确定”。
